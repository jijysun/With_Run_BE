## 🌳 브랜치 전략

### 브랜치 구조

- FE, BE 레포지토리 분리
- `main`: 배포용 브랜치
- `develop`: 테스트 및 개발 브랜치 → PR용 브랜치
- 개별 브랜치: 기능별로 생성

### 패키지 구조

- **도메인형**: 기능별 패키지 분리
  


## 📐 코딩 컨벤션

### 커밋 스타일

- `[CREATE]` : API 첫 생성, 기능 추가
- `[UPDATE]` : 버그 및 API 수정, 추가 구현 중
- `[REFAC]` : 오타 수정, 디렉토리 변경 등 소규모 수정
- `[CHORE]` : 코드 변경 없는 환경설정 등 수정 (예: setup)
- `[DOCS]` : 문서 작업
- `[DEPLOY]` : 배포 관련 작업
